## 零、结构性规律（跨周期验证，v1.7 新增）

> 以下 4 条不再只是“某个周期的观察”，而是已经在 `2019→2022` 与 `2023→2026` 两轮独立样本中都出现，因此升级为：
> **结构性规律（跨周期验证）**

| 结构性规律 | 2019→2022 证据 | 2023→2026 证据 | 硬度 |
|---|---|---|---|
| 高位难续涨 | 优先池高位爬升 = `0` | 回测 FP 集中在高位项目 | ⭐⭐⭐⭐⭐ |
| 赛道先于个体 | 新公链 / GameFi 整体先抬升 | GameFi 整体转弱 / 平台币整体变强 | ⭐⭐⭐⭐⭐ |
| 入口 / 基础设施 > 泛叙事 | `UNI / AAVE` 留下，`COMP / SUSHI` 掉队 | `PENDLE / HYPE / JUP` 跑出，`GMX / DYDX` 掉队 | ⭐⭐⭐⭐ |
| 创新浪潮先抬后分层 | DeFi Summer 全员涨 → 只剩头部 | L2 / ZK 先冲 → 12-18 月后回落分层 | ⭐⭐⭐⭐ |

## 一、总定位的修改

| 项目 | 原表述 / 旧理解 | 现在表述 | 为什么改 |
|---|---|---|---|
| 系统定位 | 更像“规律库”甚至接近“定律” | **经验型研究框架**，适合筛选 / 排序 / 排错，不适合确定性预测 | Opus 反驳后，明确暴露出幸存者偏差、周期效应、样本不足、因果不清等问题 |
| 使用方式 | 默认各条规律强度差不多 | **规律要分级使用**：S/A/B/C | 不是每条都同样硬，实战里必须区分“硬规则 / 主判断 / 提醒 / 观察” |

## 二、原有 7 条主规律的修改

| 规律 | 原表述 / 旧理解 | 现在表述 | 为什么改 |
|---|---|---|---|
| 规律 1 | **DeFi / 交易基础设施最强模式** | **DeFi / 交易基础设施是最有解释力的赢家路径之一** | 增强层把幸存者偏差打出来了：PENDLE/HYPE/JUP 很强，但 GMX/DYDX/SUSHI/COMP/BAL 这些失败样本不能忽略，所以它更像“高赔率模板”，不是“高胜率模板” |
| 规律 2 | 平台币映射交易所质量，因此长期成立 | **平台币是母交易所经营质量的高弹性映射，但波动远比想象中大** | BGB/GT/OKB/KCS 的路径都证明：平台币不是平滑上升，而是大幅波动，只能做**二阶筛选模板** |
| 规律 3 | 黄金锚定资产高度可预测 | **黄金锚定资产长期方向可预测，但中间路径波动很大** | PAXG 等样本证明：长期方向没错，但中间会被 crypto 高 beta 资产明显挤压，不能说成平滑上升 |
| 规律 4A | 新公链生态爆发路径有效 | **新公链生态爆发路径仍有效，但成功依赖三重匹配** | 规律没推翻，但样本很少，当前主要靠 SUI 支撑，所以必须加上“不能乱套模板”的限制 |
| 规律 4B | 技术叙事先行型资产通常在 12-18 个月进入回落段 | **创新先行型资产，在第一次大爆发后会快速从“全员都像赢家”进入“只剩少数入口型/基础设施型仍能维持”的分层阶段；技术叙事型只是最典型子类** | 2019→2022 的 `COMP / SUSHI / YFI` 衰退，与 2023→2026 的 `STRK / INJ / TIA` 回落，体现的是同一个母时间结构 |
| 规律 5 | 收益型 + 机构型 + 龙头型 RWA 形成结构性上升通道 | **不改核心方向，但强度下调为重点假说 / 中等强度规律** | ONDO/BUIDL 样本太少，且对宏观利率敏感，不能当硬规律 |
| 规律 6 | 纯叙事 / 名人 / 事件驱动的上涨通常不可持续 | **纯一次性事件 / 名人 / 概念驱动，且缺乏兑现的跃升通常不可持续** | 关键修正是边界：不是“叙事型都不行”，而是“只有叙事、没有兑现的才最危险”；WLD/NFT 等被移出铁板反例 |
| 规律 7 | 供给差的币不容易涨 | **供给更像决定上限，而不是决定能不能涨** | 增强层最重要修正之一：WLD/ENA 证明供给差不一定阻止上涨，但会限制它能冲多高 |

## 三、样本和标签层的修正

| 项目 | 原表述 / 旧理解 | 现在表述 | 为什么改 |
|---|---|---|---|
| DCR | 长期结构性强势 | **更像短期事件强势 / 异常反弹** | 路径 `#109 -> #175 -> #176 -> #71` 不符合“长期结构性上升” |
| WLD | 被算进“纯叙事不可持续”样本 | **灰区样本，不再当铁板反例** | 它没出局，也不是完全没产品；更像“高融资 + 强叙事 + 供给差”的灰区案例 |
| NFT/APENFT | 被算进纯叙事快速失败 | **移出该类典型样本** | 路径并不支持“快速归零式失败” |
| RAY | 常被当规律 1 核心代表 | **代表性下降** | 规律 1 的更强代表现在是 PENDLE / HYPE / JUP / MORPHO |

## 五、资本结构观察层（新增增强层，不并入主规律）

| 项目 | 原表述 / 旧理解 | 现在表述 | 为什么改 |
|---|---|---|---|
| 资本 / VC 影响 | 以前基本没单独纳入规律系统 | **资本最有价值的时候，是在加速已经成立的产品；资本最危险的时候，是在替代尚未成立的产品** | 这轮资本地图研究证明：不能只看“谁投了”，要看资本是否造成供给污染、激励错位、估值前置 |
| 低融资 vs 高融资 | 容易滑成“低融资更好 / VC 不好” | **高融资本身不是原罪，关键在于资本是加速产品还是替代产品** | SUI 是高融资赢家；问题不在融资多，而在“高融资 + 低流通 + 高预期 + 弱兑现”这个组合 |
| 一级 vs 二级资本 | 以前混着看 | **一级 VC 融资 和 二级市场机构承接 要分开看** | TAO（零一级污染 + 强二级承接）和 STRK（重一级融资 + 弱二级承接）是完全不同的信号 |
| 新 Hard fail：反身性闭环飞轮 | 旧 Hard fail 里没有这类风险 | **当代币需求过度依赖协议自身飞轮、平台资产负债表或内部流动性支撑时，需单独亮红灯** | LUNA / FTT 说明：它可能在上升阶段最强，却在飞轮反转时最脆弱 |

## 六、当前最实用的最终版（压缩版）

### 现在最关键的 6 句
1. **这不是定律库，是经验型研究框架。**
2. **规律不是一样硬，必须分级使用。**
3. **供给更像决定上限，而不是决定能不能涨。**
4. **赛道有集体命运，先选赛道，再挑项目。**
5. **创新先行型资产，第一次大爆发后默认会快速分层；技术叙事只是最典型子类。**
6. **资本要看是在加速产品，还是在替代产品。**
7. **遇到反身性闭环飞轮，要单独做 Hard fail 检查。**

## 七、一句话总总结

> **这套系统最大的升级，不是多了几条“新规律”，而是从“总结赢家像什么”，升级成了“知道哪些规则更硬、哪些只是提醒、哪些必须用供给结构和资本结构去修正”。**
