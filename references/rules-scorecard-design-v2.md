# 规律打分表设计 v2
## 给 Step 4 用的“规律评分卡”，不是把 17 条规律硬拼成平均分

> 设计目标：
>
> **把当前规律系统变成一个可重复使用的打分表，用来筛当前 top1000 / 当前候选；它服务的是“可见市场筛选”，不是 pre-TGE 黑马预测。**
>
> 核心原则：
> 1. **先过事实层，再打规律分**
> 2. **不是每条规律都适合直接打分**
> 3. **S / A 级规律做主分项，B / C 级规律更多做提醒或封顶条件**
>
> 对照依据：
> - `references/crypto-strong-coin-framework-v1.6.md`
> - `references/rules-priority-grading-v1.5.md`
> - `references/capital-structure-observation-v1.5.md`
>
> 适用对象：
> - 当前 top1000 初筛
> - Step 4 候选复核
> - 给 Opus / Amy 统一口径用

# 一、为什么不能直接把 17 条规律做成 17 个平权分项？

因为这 17 条里混了 3 种东西：

1. **可直接项目化打分的规律**
   - 例如：供给上限、赛道命运、基础设施模板、二段加速

2. **只能做负面筛查 / 风险惩罚的规律**
   - 例如：技术叙事 12-18 个月回落、纯叙事缺兑现、中继失败

3. **更像群体现象 / 市场背景提醒的规律**
   - 例如：top100 存活率 45%、top30 哑铃分布、连续加速型样本为零

所以正确做法不是：
- 17 条一人一票

而是：
- **先事实层过门槛**
- **再用可项目化规律打加减分**
- **最后用观察类规律做警报标签 / 封顶条件**

# 二、推荐的总结构

## Step A：事实层门槛（必须先过）
沿用 Step 4 原框架，6 条，0/1 打分：

1. 当前排名仍有明显上行空间
2. 近 6 个月排名有改善迹象
3. 相对 BTC 偏强
4. 相对同赛道中位数偏强
5. 供给结构健康
6. 有基本生态 / 使用证据

### 通过线
- **事实层 >= 3/6 才进入规律打分层**
- **事实层 < 3/6 直接判“暂不成立 / 预跟踪”**

---

## Step B：规律打分层（主分层）
只拿**能项目化**的规律打分。

### 建议总分
- **规律主分：-16 ~ +18**
- **资本结构加成：-4 ~ +4**
- **总分：-20 ~ +22**

---

## Step C：警报标签层（不直接加分，但会封顶/降级）
这层不做主分，只做：
- 降级提醒
- 封顶条件
- 风险标签

例如：
- top100 新进者 45% 存活率
- top30 哑铃分布
- 连续加速型样本为零
- 2024 明星资产回落
- 已在高位的强势项目续涨难度

# 三、主打分表（可直接复制到 Excel）

## 1. 赛道命运（新规律 C）
**权重：高**

| 条件 | 分数 |
|---|---:|
| 所在赛道明显处于主线 / 强支线，且项目是赛道核心受益者 | +4 |
| 赛道中性，但项目仍有独立优势 | +1 |
| 赛道明显转弱 / 逆风 | -4 |

说明：
- 这条是 S 级规律，优先级非常高
- 用来回答：**你是不是在逆赛道赌个体？**

---

## 2. 供给上限（规律 7 修正版）
**权重：最高**

| 条件 | 分数 |
|---|---:|
| 流通 >=30%，未来 12 个月解锁 <= 当前流通盘 20%，结构较干净 | +4 |
| 流通一般 / 解锁中性 | +1 |
| 流通偏低或未来 12 个月解锁压力明显 | -2 |
| 流通很差、低流通高 FDV、近端解锁重 | -4 |

说明：
- 这是 S 级规律
- 它不决定“能不能涨”，但非常影响**能涨到哪里**

---

## 3. 基础设施 / 产品驱动模板（规律 1）
**权重：高**

| 条件 | 分数 |
|---|---:|
| 明确属于 DeFi / 交易 / 金融基础设施，且产品证据强 | +3 |
| 有部分产品驱动特征，但模板不纯 | +1 |
| 不属于该模板 | 0 |

说明：
- 规律 1 不再当“铁律”
- 但它仍然是最强的**正向模板加分项之一**

---

## 4. 新公链生态爆发匹配（规律 4A）
**权重：中高**

| 条件 | 分数 |
|---|---:|
| 新链 / 新生态同时满足用户、应用、TVL 三重匹配 | +3 |
| 只满足其中一部分，仍在验证中 | 0 |
| 更像重融资新链、产品和生态没起来 | -3 |

说明：
- 不是看到新链就加分
- 这条只对**真正有生态爆发证据**的新链相关标的成立

---

## 5. 二段加速窗口
**权重：中高**

| 条件 | 分数 |
|---|---:|
| 刚进入可见区（通常 100~300 左右）且基本面同步加强 | +3 |
| 有点像，但证据还不够 | +1 |
| 已经明显不是二段窗口 | 0 |

说明：
- 这条很适合 Step 4 用
- 本质是捕捉“刚被市场重新定价”的位置

---

## 6. 中继失败（负向）
**权重：中高（惩罚项）**

| 条件 | 分数 |
|---|---:|
| 尚未出现中继失败迹象 | 0 |
| top100 附近横盘过久、迟迟无法上移 | -2 |
| 明显进入中继失败阶段 | -4 |

说明：
- 这是很实用的降级项
- 进入 top100 不等于成功，关键是能否继续上移

---

## 7. 创新先行型资产分层/回落计时器（规律 4B，负向）
**权重：高（惩罚项）**

| 条件 | 分数 |
|---|---:|
| 不属于创新先行型资产，或仍在第一次爆发早期 | 0 |
| 已进入第一次大爆发后的高风险分层窗口 | -3 |
| 已出现“只剩少数入口型/基础设施型仍能维持，其余开始回落”的典型特征 | -5 |

说明：
- 这是 S 级负向规则
- 它不再只指技术叙事；**技术叙事型（如 ZK / L2）只是最典型子类**
- DeFi 创新型资产（如 DeFi Summer 后的分层）也遵循同一母时间结构
- 很适合做“别追第一次大爆发尾段”的过滤器

---

## 8. 纯叙事无兑现（规律 6，负向）
**权重：中高（惩罚项）**

| 条件 | 分数 |
|---|---:|

| 叙事成分很高，但仍有部分兑现 | -2 |
| 纯一次性叙事 / 概念驱动，缺乏兑现 | -5 |

说明：
- 这条是排除项，不是主加分项

| 条件 | 分数 |

| 只具备其中一部分 | +1 |

- 这条不够硬，不宜给太高权重

## 10. 平台币二阶筛选（规律 2 + 平台币模板）

# 四、资本结构加成层（单独一列，不混入主规律）

| 条件 | 分数 |
|---|---:|
| 资本明显在加速已成立产品 | +2 |
| 资本与产品同步 | +1 |
| 资本中性 | 0 |
| 资本更像在替代产品 / 造成供给污染 | -2 |
| 重融资 + 低流通 + 高预期 + 弱兑现 | -4 |

说明：
- 这层来自 `capital-structure-observation-v1.5.md`
- 不替代基本面，只做修正项

### 市场温度修正项（执行门，占位）
这层**不并入项目质量总分**。

它不回答：
- 这个项目本身好不好

它只回答：
- **在当前市场周期里，评分卡给出的正向信号，能不能转成实际建仓动作**

| 市场温度 | 触发参考（占位） | 对评分卡正向信号的作用 |
|---|---|---|
| **上行 / 复苏** | BTC 站上 200 日均线，且 200 日均线走平或上拐 | `高优先级 / 观察级` 可进入执行候选 |
| **中性 / 过渡** | BTC 在 200 日均线附近反复拉锯 | 正向信号只做观察或轻仓验证 |
| **下行 / 横盘弱势** | BTC 明显位于 200 日均线下方 | 正向信号默认只跟踪、不直接建仓；负向筛选继续有效 |

说明：
- **评分卡负责项目质量，市场温度负责执行时机**
- 它不是新的项目加减分项，而是一个**执行过滤层**
- 现在先留占位；以后如果要系统化，可把 BTC vs 200D MA 作为最小可执行代理指标

# 五、警报标签层（不加总分，但会封顶）

这些不建议直接加进总分，而是做成标签：

| 标签 | 触发条件 | 作用 |
|---|---|---|
| **top100 生存警报** | 刚进 top100 | 默认提醒：别把“进 top100”当成功 |
| **哑铃分布警报** | thesis 依赖“慢慢从中段爬进 top30” | 默认压低预期 |
| **连续加速幻觉警报** | 预期里暗含“未来还会线性加速” | 提醒降温 |
| **周期回落警报** | 属于 2024 明星叙事 / 热门赛道尾段 | 默认更谨慎 |
| **供给红灯** | 低流通 + 近端大解锁 | 封顶：最高只能观察级 |
| **高位续涨折扣** | 2024 打分时已在 top50 | 最终分 ×0.7，或保守封顶到观察级 |

# 六、最终分档规则

## Step 1：先看事实层
- **事实层 < 3/6** → 直接 **暂不成立 / 预跟踪**
- **事实层 >= 3/6** → 才进入规律分层

## Step 2：再看“规律主分 + 资本加成”
### 推荐阈值
- **总分 >= 8，且事实层 >= 4** → **高优先级讨论 / 高优先级观察**
- **总分 4~7，且事实层 >= 3** → **观察级**
- **总分 <= 3** → **暂不成立 / 仅保留观察**

## Step 3：最后看封顶标签
如果触发这些标签，不能升太高：
- **供给红灯** → 最高封顶到 **观察级**
- **高位续涨折扣** → 默认把最终分乘以 `0.7`；若 thesis 主要依赖继续上冲，可直接封顶到 **观察级**
- **技术叙事回落计时器命中** → 高度谨慎，通常不进高优先级
- **Hard fail** → 直接出局

## Step 4：最后叠加市场温度（不改项目评级，只改执行级别）
- **上行 / 复苏**：`高优先级 / 观察级` 可以进入执行候选
- **中性 / 过渡**：默认更保守，优先观察、轻仓、等确认
- **下行 / 横盘弱势**：`高优先级 / 观察级` 默认只做跟踪，不直接把评分结果当买入指令

> **观察级以上 ≠ 可以直接买。**
> 真正的用法是：先用评分卡排除暂不成立，再用市场温度决定正向信号要不要执行。

# 八、已知盲区、局限与补充工具

## 1) 评分卡覆盖范围
- 这张卡主要覆盖：**已进入 top1000 的可见市场候选**
- 它擅长做：**先排错，再从剩余候选里挑强项**

## 2) 当前不覆盖的对象
- pre-TGE 项目
- 当期仍在池外的新物种
- 还没形成价格 / 排名 / 流通结构的早期协议

## 3) 已知局限：选优能力受市场温度影响很大
窗口 A（2024→2026，可见市场）与窗口 B（2022→2024，可见市场）对比后，问题已经很清楚：

| 指标 | 窗口 A | 窗口 B |
|---|---:|---:|
| 命中率 | `68.2%` | `68.4%` |
| TP | `9` | `2` |
| FP | `3` | `12` |
| Precision | `75.0%` | `14.3%` |

这说明：
- **排错能力** 在不同周期里相对稳定
- **选优能力** 强烈依赖市场环境
- 在下行 / 横盘周期，即使项目本身是好项目，也可能长期不涨

换句话说：
> **评分卡会告诉你“这项目值不值得留在场上继续看”，但不会单独保证“现在买它就会涨”。**

## 4) 系统级修正：评分卡不能单独充当买入信号
正确用法应改为两步：
1. **先用评分卡排除暂不成立的对象**
2. **再用市场温度决定观察级以上的对象要不要执行**

所以：
- `观察级以上` 不再等于 `可以买`
- `观察级以上 + 市场温度支持` 才进入执行候选

## 5) 现有补充工具
- **发现阶段**：`references/core-capital-crosscheck-v1.md`（六家资本方交叉比对 v1）
- **筛选阶段**：本评分卡（Step 4）
- **执行阶段（占位）**：市场温度修正层

换句话说：
> **资本交叉比对负责发现，评分卡负责筛选，市场温度负责执行。**

## 6) 未来待建
- 当积累到足够多的历史样本后，再单独设计：**pre-TGE 评分卡**
- 当执行层信号积累够多后，再把 **市场温度修正项** 做成正式模块
